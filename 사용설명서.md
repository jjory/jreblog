## 1️⃣ 시스템 구조 이해하기

### 🏢 사무실 공용 모드 구조

```
       ┌─────────────────────────────┐
       │     [서버 PC] 사무실 1대     │
       │  ┌───────────────────────┐  │
       │  │ • 프로그램 설치       │  │
       │  │ • API 키 보관         │  │
       │  │ • 항상 켜둠           │  │
       │  └───────────────────────┘  │
       └──────────┬──────────────────┘
                  │
            사무실 와이파이/유선랜
                  │
       ┌──────────┼──────────┬──────────┐
       ▼          ▼          ▼          ▼
   ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
   │ 직원A  │ │ 직원B  │ │ 직원C  │ │ 사장님 │
   │ PC     │ │ PC     │ │ PC     │ │ PC     │
   │ (브라우저) │ (브라우저) │ (브라우저) │ (브라우저) │
   └────────┘ └────────┘ └────────┘ └────────┘
```

### 👥 역할 분담

| 역할 | 누가 | 무엇을 함 |
|------|------|----------|
| **서버 PC** | 사장님 또는 IT 담당자 | 한 번 설치하고 매일 아침 프로그램 실행 |
| **직원 PC** | 영업 직원들 | 브라우저에서 접속해서 사용만 함. **아무것도 설치 X** |

### 💡 왜 이렇게 만드나요?

- **API 키 1개로 통합** → 청구서 1장, 비용 관리 편함
- **직원 PC에는 설치 불필요** → 직원 교체 시에도 추가 작업 없음
- **공용 비밀번호로 보안** → 외부인 접속 차단

### 🎒 무엇이 필요한가요?

**서버 PC (한 대)**
- 💻 Windows 10/11
- 🌐 인터넷 (사무실 와이파이 또는 유선랜)
- 🌐 사무실 네트워크 IP (대부분 자동)
- 💳 신용카드 (Anthropic API 결제용)

**직원 PC (여러 대)**
- 💻 Windows (또는 Mac/스마트폰도 가능)
- 🌐 같은 사무실 와이파이/유선랜
- 🌐 브라우저 (Chrome, Edge 등)
- ❌ **그 외 아무것도 필요 없음**

### 💰 비용

| 항목 | 금액 |
|------|------|
| 프로그램 자체 | 무료 |
| 블로그 글 1건 생성 | 약 170~200원 |
| 월 50건 발행 (5명이 나눠 작성) | 약 8,000~10,000원 |

## 2️⃣ 서버 PC에서 첫 설치 (한 번만)

여기서부터는 **사무실 서버 역할을 할 PC 1대에서만** 진행합니다.

### 단계 ① — Python 설치

Python은 이 프로그램을 돌릴 수 있게 해주는 엔진입니다.

1. https://www.python.org/downloads/ 접속
2. 노란색 큰 버튼 **"Download Python 3.x.x"** 클릭
3. 다운로드 된 `.exe` 파일 더블클릭
4. ⚠️ **매우 매우 중요**:
   첫 화면 아래쪽의 **"Add Python to PATH"** 체크박스를 **반드시 체크**!
   - 체크 안 하고 설치하면 작동 안 합니다. 다시 설치하세요.
5. **"Install Now"** 클릭
6. 설치 완료

### 단계 ② — Anthropic API 키 받기

1. https://console.anthropic.com 접속
2. **"Sign up"** 클릭 → 이메일 또는 Google 계정으로 가입
3. 로그인 후 왼쪽 메뉴에서 **"API Keys"** 클릭
4. **"Create Key"** 버튼 클릭
5. 키 이름 입력 (예: `office-blog`) → **Create**
6. 화면에 나오는 키 (`sk-ant-...` 로 시작)를 **복사**해서 메모장에 저장
   - ⚠️ **이 화면을 닫으면 키를 다시 볼 수 없습니다!**

7. **결제 카드 등록**:
   - 왼쪽 메뉴 → **"Billing"** → **"Add credit/debit card"**
   - 카드 정보 입력 후 최소 $5 (약 6,500원) 충전

### 단계 ③ — 프로그램 폴더 준비

이 사무실은 프로그램을 Google Drive 공유 폴더에 두고 사용합니다.

1. 받으신 `property_blog.zip` 파일을 다음 위치에 둡니다:
   ```
   G:\내 드라이브\0.사내공유\1.부동산_공유\블로그자동작성\
   ```
2. `property_blog.zip` 파일을 우클릭 → **"압축 풀기"**
3. 압축을 풀면 다음 폴더가 생깁니다 (이 폴더가 앞으로 계속 사용할 폴더):
   ```
   G:\내 드라이브\0.사내공유\1.부동산_공유\블로그자동작성\property_blog
   ```

> 📌 이 문서에서 앞으로 **`property_blog` 폴더**라고 하면 위 경로의 폴더를 말합니다.

> ⚠️ **Google Drive 사용 시 알아둘 점**
> - 이 폴더는 사내 공유 폴더라 다른 직원도 파일을 볼 수 있습니다.
> - 프로그램을 **실행하는 PC는 한 대로 정해두세요** (서버 PC). 여러 PC에서 동시에
>   `시작하기.bat` 을 실행하면 충돌이 날 수 있습니다.
> - `pip install` 같은 무거운 작업 중에는 Google Drive 동기화가 잠시 느려질 수 있으나,
>   작업 자체에는 문제없습니다.
> - 직원들은 이 폴더를 직접 열 필요 없이, 서버 PC가 띄운 화면에 **브라우저로 접속**만
>   하면 됩니다 (3번 챕터 참고).

### 단계 ④ — 자동 설치 실행

1. `property_blog` 폴더 안의 **`설치하기.bat`** 파일을 더블클릭
2. 검은 창이 뜨고 자동으로 부품 설치 진행
3. 마지막에 **"설치 완료!"** 메시지가 나오면 성공
4. 아무 키 눌러서 창 닫기

> ⚠️ **Windows Defender 경고가 뜨면**: "추가 정보" 클릭 → "실행" 클릭

### 단계 ⑤ — API 키와 공용 비밀번호 설정

1. `property_blog` 폴더 안에서 **`.env.example`** 파일 찾기
2. 이 파일을 복사한 후 (Ctrl+C), 같은 자리에 붙여넣기 (Ctrl+V)
3. 복사된 파일 이름을 **`.env`** 로 변경
   - 원본: `.env.example` → 복사본 이름: `.env`
   - ⚠️ `.example` 부분만 지우세요 (확장자 보이게 하려면 폴더 위쪽 "보기" → "파일 확장명" 체크)
4. `.env` 파일을 우클릭 → **"연결 프로그램" → "메모장"** 으로 열기
5. 다음 부분들을 본인 정보로 수정:

```
# Claude API 키 (위에서 받은 진짜 키로 교체)
ANTHROPIC_API_KEY=sk-ant-실제로받은긴키...

# Gemini API 키 (무료 — 선택사항이지만 강력 추천)
# 발급: https://aistudio.google.com/apikey
# 있으면 도면 분석이 대부분 무료로 처리됨 (하이브리드 모드)
GEMINI_API_KEY=AIzaSy_실제로받은키...

# 사무실 공용 비밀번호 (직원들에게 알려줄 비밀번호)
OFFICE_PASSWORD=japanreal2026

# 회사 정보
COMPANY_NAME=JRE일본부동산
KAKAO_TALK_ID=japanreal2
COMPANY_PHONE=070-8201-5740
```

> 💡 **Gemini API 키는 무료입니다**. https://aistudio.google.com/apikey 에서 Google 계정으로 로그인 → "Create API key" 클릭만 하면 발급됩니다. 카드 등록 불필요. 이 키가 있으면 도면 분석의 대부분이 무료로 처리되고, 어려운 케이스만 Claude(유료)가 자동 재시도합니다.

6. **저장 후 닫기** (Ctrl+S)

### 단계 ⑥ — Windows 방화벽 설정 (직원 PC가 접속할 수 있게)

직원 PC가 서버 PC에 접속할 수 있게 방화벽을 한 번만 열어줍니다.

1. `property_blog` 폴더 안의 **`방화벽허용하기.bat`** 파일을 **우클릭**
2. **"관리자 권한으로 실행"** 클릭
3. "예" 클릭
4. 자동으로 방화벽 규칙이 추가됨
5. **"방화벽 설정 완료!"** 메시지 확인

> 💡 한 번만 실행하면 됩니다. 다음부터는 안 해도 돼요.

### 단계 ⑦ — 첫 실행 테스트

1. `시작하기.bat` 더블클릭
2. 검은 창에 다음과 같이 표시됨:
   ```
   [이 PC에서 사용할 때]
     브라우저에서:  http://localhost:8501

   [다른 직원 PC에서 사용할 때]
     브라우저에서:  http://192.168.x.x:8501
   ```
3. **위의 두 번째 주소(192.168.x.x:8501) 를 메모하세요**. 이걸 직원들에게 알려줄 거예요.
4. 잠시 후 서버 PC의 브라우저가 자동으로 열림
5. 로그인 화면에서 사무실 비밀번호 입력
6. 잘 들어가지면 성공!

✅ **첫 설치 완료!**

## 3️⃣ 직원 PC에서 접속하기

### 직원들에게 공유할 정보 (카카오톡으로 보내세요)

```
📢 사무실 부동산 블로그 시스템 사용 안내

1. 사무실 와이파이/유선랜에 연결되어 있는지 확인
2. 브라우저(크롬, 엣지)에서 접속:
   http://192.168.x.x:8501  ← 사장님이 알려주신 주소
3. 사무실 비밀번호 입력
4. 끝! 매물 도면을 올려서 사용하세요
```

### 즐겨찾기 추가 (직원이 한 번만)

1. 위 주소에 접속한 후 브라우저에서 **Ctrl+D**
2. 이름: `부동산블로그시스템` 으로 저장
3. 다음부터는 즐겨찾기에서 클릭만 하면 됨

### 모바일/태블릿에서도 가능

같은 사무실 와이파이에 연결돼 있다면 **스마트폰 브라우저에서도 동일한 주소로 접속 가능**합니다.

## 4️⃣ 매일 사용하는 방법

### 🌅 매일 아침 (서버 PC 담당자)

1. `시작하기.bat` 더블클릭
2. 검은 창이 그대로 떠 있도록 둠 (창 닫지 말기!)
3. 끝

### 직원 매일 작업 (PC 또는 모바일)

1. 브라우저 즐겨찾기에서 `부동산블로그시스템` 클릭
2. 이름 + 비밀번호 입력
3. 사용 시작

### 🌙 매일 저녁 (서버 PC 담당자)

검은 창에서 **`Ctrl + C`** 누르면 종료. 검은 창 닫기.

(또는 그냥 PC를 끄셔도 됩니다.)

---

### 📝 매물 1건 작성하는 방법 (5분 워크플로)

화면에 4개 탭이 있습니다. 순서대로 진행하세요.

#### 🎯 사이드바 설정 (왼쪽)

- **타깃 비자**: 글의 톤 참고용 (워홀/유학/취업/경영관리/영주권). 추천 이유는 비자별로 나누지 않고 통합 작성됩니다.
- **글 스타일**: 친근형/격식형/간결형/스토리텔링형/분석형
- **환율**: 100엔당 원화. 가끔 갱신.
- **AI 모델**:
  - `claude-opus-4-7`: 가장 정확. **추천**
  - `claude-sonnet-4-6`: 더 빠르고 1/5 가격

#### 1️⃣ 첫 번째 탭: 도면 업로드 (한 번에 최대 5개)

1. **"Browse files"** 클릭 → 마이소크 파일 선택 (**한 번에 최대 5개**)
2. 미리보기 확인
3. **"🔍 전체 도면 분석 시작"** 버튼 클릭
4. 파일 수에 따라 대기 (1개당 15~30초)

💡 업로드 팁:
- **지원 형식: JPG · PNG · WEBP · GIF · PDF** (PDF도 바로 업로드 가능)
- 어두운 사진은 인식률 ↓
- 회전된 사진은 정확도 ↓ → 똑바로 세워서 올리세요
- 한 번에 5개를 올리면 블로그 글 5개가 한꺼번에 만들어집니다

#### 2️⃣ 두 번째 탭: 추출 결과 검토 ⭐ **중요**

AI가 도면에서 뽑아낸 정보가 매물별로 표시됩니다.

**🚨 여기서 반드시 사람이 한 번 확인.** AI가 90~95%는 정확하지만 100%는 아닙니다.

꼭 확인할 항목:
- 월세, 관리비 (만엔 표기 헷갈릴 수 있음)
- 외국인 입주 가능 여부
- 보증금, 사례금
- 역까지 도보 분

틀린 부분은 JSON 박스에서 직접 수정. 예:
```
"foreigners_ok": null   ← 이걸
"foreigners_ok": true   ← 이렇게 수정
```

다 됐으면 **"✍️ 한국어 블로그 글 생성"** 클릭.

#### 3️⃣ 세 번째 탭: 블로그 미리보기

생성된 글 확인. 마음에 들면 **"💾 HTML 다운로드"** 클릭.

> 💡 직원 PC에서 다운로드하면 그 직원 PC의 다운로드 폴더로 저장됨.

#### 4️⃣ 네이버에 발행 — 방식 A (수동, 즉시 가능)

1. 다운로드한 HTML 파일을 메모장으로 열기
2. **모두 선택 + 복사** (Ctrl+A → Ctrl+C)
3. 네이버 블로그 → **글쓰기**
4. 글쓰기 창 우측 상단의 **"…" → "HTML 편집"** 클릭
5. 빈 칸에 붙여넣기 (Ctrl+V)
6. 다시 일반 편집 모드로 돌아가기
7. **제목 입력** (시스템이 추천한 제목 사용)
8. 미리보기로 확인 → **발행**

✅ **5분 완료!**

## 5️⃣ 네이버 자동 발행 설정 (선택)

> ⚠️ 처음에는 건너뛰세요. 익숙해진 후 도전.

### 네이버 개발자센터에서 앱 등록

1. https://developers.naver.com 접속 → 네이버 로그인
2. **"Application" → "애플리케이션 등록"**
3. 폼 작성:
   - **애플리케이션 이름**: `도쿄부동산 자동발행` (자유)
   - **사용 API**: ✅ 네이버 아이디로 로그인, ✅ 블로그 **둘 다 체크**
   - **서비스 환경**: PC 웹
   - **Callback URL**: `http://localhost:8501/callback`
4. 등록 → **Client ID** 와 **Client Secret** 받음 → 메모장에 저장
5. "네이버 아이디로 로그인" API 심사 자동 신청 (약 3일 소요)

### `.env` 파일에 네이버 키 추가

서버 PC에서 `.env` 파일 다시 열고:
```
NAVER_CLIENT_ID=받은_Client_ID
NAVER_CLIENT_SECRET=받은_Client_Secret
NAVER_REDIRECT_URI=http://localhost:8501/callback
```

저장 후 `시작하기.bat` 재실행.

### 첫 사용 시 네이버 인증

⚠️ **이 작업은 서버 PC에서만 가능합니다** (Callback URL이 localhost로 설정돼 있어서).

1. 서버 PC 브라우저에서 4번 탭으로 이동
2. 화면의 URL 클릭 → 네이버 로그인 → 권한 허용
3. 브라우저 주소창의 `code=ABCD...` 에서 `ABCD...` 부분만 복사
4. 4번 탭 입력창에 붙여넣기 → "토큰 발급" 클릭
5. ✅ 완료. 이후 모든 직원이 자동 발행 사용 가능.

## 6️⃣ 자주 묻는 질문

### Q1. 서버 PC를 매번 켜둬야 하나요?
A. 네. 직원들이 사용하는 동안엔 서버 PC와 `시작하기.bat`의 검은 창이 떠 있어야 합니다. 출근하실 때 켜고, 퇴근하실 때 끄면 됩니다.

### Q2. 서버 PC가 갑자기 꺼졌어요. 어떻게 해야 하나요?
A. 다시 켠 후 `시작하기.bat` 더블클릭. 직원들에게 "다시 접속해주세요" 카톡 보내기. 작업 중이던 데이터는 사라질 수 있으니 직원들은 중요한 매물은 자주 다운로드해두는 게 좋아요.

### Q3. 비밀번호를 바꾸고 싶어요.
A. 서버 PC의 `.env` 파일을 열어서 `OFFICE_PASSWORD=새비밀번호` 로 수정 후 저장. 검은 창에서 Ctrl+C로 종료 후 `시작하기.bat` 다시 실행. 직원들에게 새 비밀번호 공유.

### Q4. 한 직원이 그만뒀어요. 그 직원이 접속 못하게 하려면?
A. 비밀번호를 바꾸고 (Q3 참조) 남은 직원들에게만 새 비밀번호 공유.

### Q5. 두 명이 동시에 작업해도 되나요?
A. 네, 됩니다. 각자 브라우저에서 따로 작업하며 서로 영향 없습니다. 단, **AI 분석은 동시 2~3건이 한계**예요. 그 이상이면 줄 서서 처리됩니다.

### Q6. 사무실 와이파이가 아닌 외부(예: 출장 중)에서도 접속할 수 있나요?
A. 기본 설정으로는 안 됩니다 (보안). 외부 접속이 필요하면 별도 설정(VPN, ngrok 등) 필요. 개발자에게 문의.

### Q7. 마이소크가 PDF인데 올릴 수 있나요?
A. 네, **PDF도 바로 업로드 가능**합니다. JPG·PNG·WEBP·GIF·PDF 모두 지원합니다.
별도 변환 없이 그대로 올리시면 됩니다.

### Q8. AI가 정확하지 않을 때는?
A. 2번 탭의 JSON 박스에서 직접 수정합니다. 예:
- 월세 ¥85,000을 ¥850,000으로 잘못 읽음 → 0 하나 지우기
- 외국인可 표시를 못 읽음 → `null` 을 `true` 로 변경

### Q9. 매물 1건 만들 때마다 얼마인가요?
A. 약 170~200원. Anthropic 콘솔 → Billing 에서 실시간 사용량 확인. **누가 얼마나 사용했는지는 표시 안 됩니다** (필요하면 추가 개발 필요).

### Q10. 같은 매물로 워홀러용·유학생용 글을 따로 만들려면?
A. 1, 2번 탭은 그대로. 사이드바에서 **타깃 비자만 변경 후** 2번 탭에서 "블로그 글 생성" 다시 누르면 새 글 생성.

### Q11. 일본어/영어 블로그도 만들 수 있나요?
A. 현재는 한국어 + 네이버 전용. 다국어로 확장하려면 약간의 코드 수정 필요. 개발자에게 문의.

### Q12. 같은 PC에서 직원 모두가 쓰는 게 아니라 각자 본인 PC에 설치하는 게 좋지 않나요?
A. 가능은 하지만 권장하지 않습니다. 이유:
- API 키를 직원 수만큼 발급해야 함
- 비용 정산 복잡
- 직원 교체 시 새 직원 PC에 다시 설치해야 함
- 프로그램 버전 관리 번거로움

## 7️⃣ 문제가 생기면

### 🚨 가장 흔한 오류와 해결법

#### 직원 PC에서 "사이트에 접속할 수 없습니다"
**원인 1**: 서버 PC가 꺼져있거나 시작하기.bat이 안 떠 있음
→ 사장님께 확인 요청

**원인 2**: 사무실 와이파이가 아닌 모바일 데이터에 연결됨
→ 사무실 와이파이로 변경

**원인 3**: 방화벽 설정 안 됨
→ 서버 PC에서 `방화벽허용하기.bat` 우클릭 → 관리자 권한 실행

**원인 4**: IP 주소가 변경됨
→ 서버 PC 검은 창에 표시된 새 IP 주소를 직원들에게 공유

#### "Python이 설치되어 있지 않습니다"
→ 단계 ① 다시. "Add Python to PATH" 체크 잊지 마세요.

#### "AuthenticationError: Invalid API key"
→ `.env` 파일의 ANTHROPIC_API_KEY 확인. 키 앞뒤 공백 없는지, 전체 다 복사했는지.

#### "Rate limit exceeded"
→ 너무 많이 동시에 사용. 1~2분 기다린 후 다시.

#### "Insufficient credits"
→ Anthropic 잔액 부족. console.anthropic.com → Billing 에서 충전.

#### 서버 PC 검은 창이 갑자기 닫혔어요
→ 그냥 다시 `시작하기.bat` 더블클릭.

#### IP 주소가 매일 바뀌어서 직원들이 헷갈려해요
→ 서버 PC에 고정 IP 설정 권장. 인터넷 검색 "Windows 고정 IP 설정" 또는 IT 담당자에게 문의.

#### "포트 8501이 이미 사용 중입니다"
→ 검은 창에서 Ctrl+C로 종료한 후 1분 대기 후 다시 시작. 또는 PC 재부팅.

### 🆘 그래도 해결 안 되면

1. 서버 PC의 검은 창에 빨간 글씨로 표시된 **에러 메시지를 그대로 캡처**
2. 다음 중 하나에 문의:
   - 개발자에게 카톡/이메일로 전달
   - ChatGPT, Claude.ai 에 "이 에러 어떻게 해결?" 문의
