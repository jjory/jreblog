# 🎨 블로그 양식 변경 가이드

블로그 글의 톤·구조·표현을 바꾸는 4가지 방법을 난이도 순으로 안내합니다.

---

## 📋 4가지 변경 레벨 한눈에 보기

| 레벨 | 방법 | 난이도 | 영향 범위 | 적용 시점 |
|------|------|--------|----------|----------|
| **레벨 1** | UI 드롭다운에서 스타일 선택 | ⭐ | 매번 선택 | 즉시 |
| **레벨 2** | "이번 글 특별 지시" 입력 | ⭐ | 일회성 | 즉시 |
| **레벨 3** | `styles/` 폴더 파일 편집 | ⭐⭐ | 영구 | 다음 글부터 |
| **레벨 4** | `src/generator.py` 프롬프트 수정 | ⭐⭐⭐ | 전체 구조 | 다음 글부터 |

**👉 추천 순서**: 레벨 1 → 부족하면 레벨 2 → 자주 쓰는 패턴이면 레벨 3 → 구조 자체를 바꾸고 싶으면 레벨 4

---

## 🎯 레벨 1: 스타일 드롭다운 (가장 간단)

### 사용법

앱 왼쪽 사이드바 → **📝 글 스타일** 드롭다운에서 선택.

### 기본 제공 5가지 스타일

| 스타일 | 톤 | 추천 대상 |
|--------|------|----------|
| **친근형** (기본) | 따뜻하고 친근, 이모지 적극 | 워홀러, 유학생, 20~30대 |
| **격식형** | 정중·전문적, 합쇼체 | 경영관리 비자, 법인계약, 임원 |
| **간결형** | 핵심만 짧게, 표 위주 | 일본 부동산 경험자, 재계약자 |
| **스토리텔링형** | 감성적, 라이프스타일 묘사 | 여성 손님, 첫 도쿄 거주자 |
| **분석형** | 데이터·비교 중심, 강점/약점 분석 | 신중한 가족 단위, 장기 거주 |

### 같은 매물로 여러 스타일 비교

1. 1번 탭에서 마이소크 분석 (1회)
2. 2번 탭에서 데이터 검토
3. **사이드바에서 스타일 변경 → 2번 탭에서 "블로그 글 생성" 클릭**
4. 3번 탭에서 결과 확인
5. 다시 사이드바에서 다른 스타일 → 다시 생성

> 💡 매물 분석은 비싸지만 블로그 생성은 저렴 (글 1건당 약 150원). 같은 매물로 3~4개 스타일 비교 시도해보고 가장 좋은 걸 발행하시면 됩니다.

---

## 🎯 레벨 2: 이번 글 특별 지시 (일회성)

### 사용법

2번 탭의 **"✍️ 블로그 글 생성"** 버튼 위에 있는 텍스트박스에 자유롭게 입력.

이 지시사항은 **이번 글에만** 적용되고, 다른 매물에는 영향 없습니다.

### 자주 쓰는 패턴

#### 매물 특성 강조
```
가구·가전 풀옵션 포함되어 있으니 그 부분을 중점적으로 강조해주세요.
이사 비용 절감 측면도 언급.
```

```
여성 손님 대상이므로 안전성 (오토락, 모니터 인터폰, 야간 귀가 동선) 강조.
주변 한인 미용실, 한인 교회 정보도 한 줄 언급.
```

```
1층 매물이지만 보안창과 셔터가 있다는 점을 강점으로 풀어주세요.
대신 1층 단점 (소음, 사생활)에 대한 솔직한 언급도 짧게 포함.
```

#### 타깃 손님 더 좁히기
```
30대 후반~40대 초반 한국인 직장인 가족 (자녀 1명) 대상.
유치원·초등학교 거리, 가족용 슈퍼마켓, 소아과 정보 비중 늘려주세요.
```

```
60대 이상 한국 영주권자 대상. 큰 글씨로, 한자 표기는 최소화하고
한국어 발음을 병기해주세요. 예: 礼金 → 사례금(레이킨)
```

#### 특정 키워드·표현 강조
```
"무권리금"이라는 단어를 본문에 2~3회 자연스럽게 노출시켜 주세요 (SEO 키워드).
"신주쿠 한인타운 도보권"도 1회 이상.
```

```
이 매물은 비교적 신축이므로 "신축급", "리노베이션 완료" 등의 키워드를
제목과 첫 문단에 자연스럽게 포함시켜 주세요.
```

#### 길이·구조 조정
```
이번엔 좀 짧게 (전체 1000자 정도). 위치와 가격 위주로.
```

```
초기비용 섹션을 평소보다 더 자세히. 항목별 한국 환산 + 왜 그런 비용이 발생하는지 짧은 설명 추가.
```

### 💡 팁

- 한 글에 너무 많이 (3~4가지 이상) 지시하면 우선순위가 흐려짐. 1~2개로 압축 권장.
- 지시 내용을 메모장에 모아두고 비슷한 상황에서 재활용하면 효율적.
- 자주 쓰는 패턴은 **레벨 3 (영구 스타일 편집)** 으로 옮기세요.

## 🎯 레벨 3: 스타일 파일 편집 (영구 변경)

### 어떤 경우에 쓰나?

- 같은 지시를 매번 입력하기 귀찮을 때
- 특정 스타일 자체의 톤을 사무실 색깔로 맞추고 싶을 때
- 새로운 손님 타깃을 정기적으로 다루게 됐을 때 (예: 한국 대학생 단기 어학연수)

### 편집 방법

#### 방법 A: VSCode 사용 (추천)

1. VSCode에서 **부동산블로그** 폴더 열기
2. 왼쪽 파일 목록 → **`styles`** 폴더 → 편집할 스타일 클릭
3. 내용 수정
4. **`Ctrl + S`** 저장
5. 다음 블로그 생성 시 자동 반영 (앱 재시작 불필요)

#### 방법 B: 메모장 사용

1. `부동산블로그/styles/` 폴더 열기
2. 편집할 파일 우클릭 → **연결 프로그램 → 메모장**
3. 수정 후 저장

### 스타일 파일 형식

각 스타일 파일은 단순한 마크다운 텍스트입니다. 정해진 구조는 없고, **블로그 글을 어떻게 쓸지 자연스러운 한국어로** 적으시면 됩니다.

#### 예시: `친근형.md` 수정

**원본**:
```markdown
- 친근하고 따뜻한 톤
- "안녕하세요, 도쿄 ○○부동산입니다 😊" 식으로 시작
- 이모지를 섹션 제목에 적극 활용
```

**수정 후 (회사 색깔 추가)**:
```markdown
- 친근하고 따뜻한 톤
- "안녕하세요, 도쿄에서 한국인 부동산 전문 10년차 [김영업]입니다 😊" 식으로 시작
- 이모지를 섹션 제목에 적극 활용
- 본인 경험담 한 줄 자연스럽게 (예: "저도 처음 도쿄 왔을 때 이 동네에 살았어요")
- 회사 보유 매물 수를 마지막에 가볍게 언급 ("현재 200여 건 매물 보유")
```

저장하면 다음 글부터 모든 친근형 글이 이렇게 작성됩니다.

### 새 스타일 추가

#### 예시: "어르신용" 스타일 신규 추가

1. `styles/` 폴더에 **`어르신용.md`** 새 파일 생성
2. 다음과 같이 작성:

```markdown
# 어르신용

**대상**: 60대 이상 한국 영주권자, 자녀의 도쿄 거주 보조 검토 중인 부모

## 톤 가이드

- 큰 흐름·핵심만 천천히, 한 문장 한 정보씩
- 일본어 한자 옆에 항상 한국어 발음 병기 (예: 礼金 → 사례금(레이킨))
- 합쇼체 (~합니다, ~입니다, ~이십니다)
- 가족(자녀·손주) 입장에서 안심할 수 있는 부분 강조
- 의료 시설(병원·약국) 도보 거리 반드시 언급
- 한국 직항 공항 (나리타·하네다) 접근성 강조 (왕래 편의)

## 인사말 예시

> 안녕하십니까. 도쿄에서 한국분들 부동산 상담을 도와드리는 [회사명]입니다.
> 한국에 자주 왕래하시는 어르신 분들께도 안심하고 추천드릴 수 있는 매물 한 채를 소개해 드리겠습니다.

## 강조 사항

- 도보권 의료 시설 (특히 한국어 가능 병원)
- 공항 접근성 (한국 왕래)
- 1층·저층·엘리베이터 유무 (계단 부담)
- 주변 한식당·한인 마트
- 야간 안전성

## CTA 스타일

> 사진과 영상으로 미리 확인하실 수 있도록 카카오톡으로 자료 보내 드립니다.
> 전화 상담도 가능합니다 (한국어).
```

3. 저장
4. 앱 사이드바의 **글 스타일 드롭다운** 에 자동으로 **"어르신용"** 추가됨

> 💡 파일명이 곧 스타일 이름입니다. 한글 OK, 공백·특수문자 자제 권장.

### 스타일 임시 비활성화

스타일 파일을 지우지 않고 임시로 숨기고 싶으면, 파일명 앞에 `_` 를 붙이면 됩니다.

예: `간결형.md` → `_간결형.md` → 드롭다운에서 사라짐

## 🎯 레벨 4: 마스터 프롬프트 수정 (전체 구조 변경)

### 어떤 경우에 쓰나?

스타일로 해결 안 되는 **구조적 변경** 이 필요할 때:
- 새 섹션을 모든 글에 항상 추가 (예: "📺 주변 한국 방송 시청 가능 여부")
- 출력 형식 자체 변경 (예: JSON에 새 필드 추가)
- 절대 규칙 추가 (예: 모든 글에 면적 평수 (m²와 평) 병기)
- 출력 길이·구조 자체 변경

### 편집 위치

**파일**: `src/generator.py`

**찾을 변수**: `BLOG_GENERATION_PROMPT` (40번 줄 근처)

### 자주 하는 수정 패턴

#### 1) 새 섹션 추가

`# html_content 구성 가이드` 섹션을 찾아서 항목 추가:

```python
# html_content 구성 가이드
다음 섹션을 포함하세요...
- 인사말 + 한 줄 매물 소개
- 매물 기본정보 (표 형태 권장)
- 초기비용 총정리
- 방 구조와 설비
- 위치와 생활 인프라
- {target_visa_label}에게 추천하는 이유
- 계약 진행 절차 (외국인 전용)
- 🆕 한국 본가와의 연결 — 한국 직항 항공 / 한국 송금 편의 / 시차 등
- 상담 신청
```

#### 2) 절대 규칙 추가

`# 작성 원칙` 섹션에 번호 추가:

```python
9. **모든 면적 표기는 m²와 평(坪)을 병기**, 예: 23.5㎡ (약 7.1평)
10. **이 매물의 1년·5년 거주 시뮬레이션을 본문에 반드시 포함**
```

#### 3) 출력 JSON에 새 필드 추가

`# 출력 형식` 섹션 수정:

```python
{{
  "title": "...",
  "meta_description": "...",
  "html_content": "...",
  "hashtags": [...],
  "summary_for_chat": "...",
  "instagram_caption": "🆕 인스타그램 게시용 한 줄 요약 (이모지 포함, 100자 이내)",
  "key_selling_points": ["🆕 영업 토크용 핵심 셀링 포인트 3가지"]
}}
```

이렇게 추가하면 매물별로 인스타그램용 카피와 영업 포인트도 함께 생성됩니다.

### ⚠️ 주의사항

- 프롬프트 수정 후 반드시 **테스트 글 1~2개 생성** 해서 결과 확인
- 잘못 수정하면 JSON 파싱이 깨질 수 있음. 백업해두기:
  ```
  src/generator.py → src/generator_backup_20260513.py
  ```
- 중괄호 `{` `}` 는 프롬프트 안에서 `{{` `}}` 로 이중 표기해야 함 (Python f-string 규칙)

## 📊 실전 시나리오별 추천

### 시나리오 1: "오늘 워홀러 상담이 많아서, 워홀러용 글만 빨리 몇 개 뽑고 싶다"
→ **레벨 1**: 사이드바에서 비자=워킹홀리데이 + 스타일=친근형 → 매물별로 빠르게 생성

### 시나리오 2: "이번 매물은 일층이라 단점을 솔직하게 풀고 싶다"
→ **레벨 2**: 특별 지시에 "1층 단점도 솔직히 언급, 대신 가격·편의성 장점 부각"

### 시나리오 3: "사무실 전체 글에 본인 영업사원 사진·전화번호를 항상 마지막에 넣고 싶다"
→ **레벨 3**: 모든 스타일 파일의 "CTA 스타일" 섹션을 본인 정보로 수정

### 시나리오 4: "모든 글에 5년 거주 시뮬레이션 표를 추가하고 싶다"
→ **레벨 4**: `generator.py` 의 `# html_content 구성 가이드`에 신규 섹션 추가

### 시나리오 5: "인스타그램에도 같이 올릴 카피가 매번 필요하다"
→ **레벨 4**: 출력 JSON에 `instagram_caption` 필드 추가

### 표준 테스트 절차

1. 같은 매물 데이터 1건 준비 (`examples/sample_property_data.json` 활용 가능)
2. 변경 전 글 1개 생성 → 저장
3. 변경 적용
4. 같은 매물로 변경 후 글 1개 생성 → 저장
5. 둘을 나란히 비교

### 빠른 비교를 위한 팁

3번 탭에서 **"📋 전체 JSON 다운로드"** 버튼을 활용해 두 결과를 파일로 저장하면 비교가 쉽습니다.

## 📝 변경 후 체크리스트

스타일 변경 후 글이 잘 나오는지 확인할 5가지:

- [ ] **첫 200자**에 위치·평면도·월세·외국인 가능 여부가 모두 들어가 있는가? (SEO 핵심)
- [ ] **초기비용 한화 환산**이 정확히 계산돼 있는가?
- [ ] **타깃 비자**에게 적합한 이유가 명확한가?
- [ ] **외국인 입주 가능 여부**가 명시되어 있는가?
- [ ] **CTA (상담 연결)**이 자연스럽게 들어가 있는가?

위 5가지 중 하나라도 누락되면 → 마스터 프롬프트의 절대 규칙이 약해진 것. **레벨 4** 에서 강화 필요.
