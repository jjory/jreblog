## 1️⃣ VSCode가 뭔가요? 왜 쓰나요?

**VSCode (Visual Studio Code)** 는 Microsoft가 만든 무료 코드 편집기입니다. 메모장의 강화 버전이라고 생각하시면 돼요.

이 시스템에서 VSCode를 쓰면 좋은 점:
- **한 화면에서 모든 것**: 파일 편집 + 터미널 + 프로그램 실행 + 오류 확인
- **`.env` 파일 편집이 편리**: 색상으로 구분되어 보임
- **F5 한 번에 실행**: 더블클릭보다 편함 (설정 파일 미리 넣어드렸어요)
- **오류 메시지 확인 쉬움**: 빨간 글씨로 명확하게 표시
- **블로그 글 스타일 수정 가능**: `src/generator.py` 의 프롬프트를 편집해서 톤 변경

## 2️⃣ .bat 더블클릭 vs VSCode — 언제 쓰나?

**둘 다 같은 시스템을 다루는 도구입니다. 함께 쓰면 됩니다.**

| 상황 | 추천 도구 |
|------|----------|
| 매일 사용 (직원, 비개발자) | **`.bat` 더블클릭** ✅ |
| API 키, 비밀번호 변경 | **VSCode** ✅ |
| 회사 정보 (카톡 ID 등) 변경 | **VSCode** ✅ |
| 블로그 글 톤 변경 | **VSCode** ✅ (필수) |
| 오류 발생 시 원인 분석 | **VSCode** ✅ |
| 코드 수정·확장 | **VSCode** ✅ |
| 빠른 1회 실행 | **`.bat` 더블클릭** ✅ |

→ **사장님은 VSCode + `.bat`, 직원은 브라우저만** 이 가장 효율적인 조합입니다.

## 3️⃣ VSCode 설치

### 단계 ① — 다운로드

1. https://code.visualstudio.com 접속
2. 파란색 큰 버튼 **"Download for Windows"** 클릭
3. 다운로드된 `VSCodeUserSetup-x64-xxxx.exe` 파일 더블클릭

### 단계 ② — 설치 진행

1. 약관 동의 → **다음**
2. 설치 위치 → **다음** (기본값 그대로)
3. 시작 메뉴 폴더 → **다음**
4. ⚠️ **"추가 작업 선택" 화면이 가장 중요합니다**. 다음을 **모두 체크**하세요:

   - ☑️ 바탕화면에 바로 가기 만들기
   - ☑️ **"Code(으)로 열기" 작업을 디렉토리 컨텍스트 메뉴에 추가** ← 폴더 우클릭으로 VSCode 열기 가능
   - ☑️ **"Code(으)로 열기" 작업을 파일 컨텍스트 메뉴에 추가** ← 파일 우클릭으로 열기 가능
   - ☑️ 지원되는 파일 형식의 편집기로 Code 등록
   - ☑️ **PATH에 추가** ← 매우 중요!

5. **다음 → 설치 → 마침**

설치 완료 후 VSCode가 자동으로 실행됩니다.

## 4️⃣ 한글 메뉴 설정 (선택)

VSCode 기본 언어는 영어지만 한글로 바꿀 수 있습니다.

1. VSCode 왼쪽 사이드바에서 **정사각형 4개 모양 아이콘** 클릭 (Extensions, 또는 `Ctrl + Shift + X`)
2. 위쪽 검색창에 **"Korean"** 입력
3. **"Korean Language Pack for Visual Studio Code"** (Microsoft 제작) 선택
4. 파란색 **"Install"** 클릭
5. 우측 하단에 팝업이 뜸 → **"Change Language and Restart"** 클릭
6. VSCode가 재시작되면서 메뉴가 한글로 변경됨

> 💡 영어가 더 편하시면 건너뛰셔도 됩니다. 본 가이드는 한글 메뉴 기준으로 설명합니다.

## 5️⃣ Python 확장 설치 (필수)

이 확장이 있어야 VSCode에서 Python 프로그램을 실행할 수 있습니다.

1. 왼쪽 사이드바 → **Extensions** (`Ctrl + Shift + X`)
2. 검색창에 **"Python"** 입력
3. 첫 번째 결과 **"Python"** (Microsoft 제작, 다운로드 수가 가장 많음) 선택
4. **"Install"** 클릭
5. 설치 완료 (몇 초)

> 💡 함께 설치하면 좋은 확장 (선택):
> - **DotENV**: `.env` 파일 색상 구분 (검색: dotenv)

## 6️⃣ 프로젝트 열기

> 📌 이 사무실의 프로그램 폴더 위치:
> ```
> G:\내 드라이브\0.사내공유\1.부동산_공유\블로그자동작성\property_blog
> ```
> 앞으로 이 폴더를 **`property_blog` 폴더**라고 부릅니다.

### 방법 A: 메뉴에서 열기

1. VSCode → **파일 → 폴더 열기** (또는 `Ctrl + K`, `Ctrl + O`)
2. 다음 경로를 따라 들어가서 **`property_blog`** 폴더 선택:
   `G 드라이브 → 내 드라이브 → 0.사내공유 → 1.부동산_공유 → 블로그자동작성 → property_blog`
3. **"폴더 선택"** 클릭
4. 신뢰 여부 묻는 창 → **"예, 작성자를 신뢰합니다"**

### 방법 B: 폴더 우클릭으로 열기 (더 빠름)

1. Windows 탐색기에서 위 경로의 **`property_blog`** 폴더를 찾아 우클릭
2. **"Code(으)로 열기"** 클릭

> 위 메뉴는 VSCode 설치 시 "Code(으)로 열기" 옵션을 체크해야 보입니다.

> ⚠️ **꼭 `property_blog` 폴더 자체를 여세요.** 한 단계 위인 `블로그자동작성`
> 폴더를 열면 터미널이 엉뚱한 위치를 가리켜 오류가 납니다.

### 화면 구성 확인

## 7️⃣ Python 인터프리터 선택

VSCode가 어떤 Python을 쓸지 알려주는 단계입니다.

1. **`Ctrl + Shift + P`** 누르기 (명령 팔레트 열기)
2. 입력창에 **"Python: Select Interpreter"** 입력
3. 검색 결과에서 동일한 이름 클릭
4. Python 버전 목록이 나옴
5. 가장 위쪽의 **Python 3.x.x (Recommended)** 선택
6. 화면 오른쪽 아래에 선택된 버전이 표시됨

> ⚠️ **Python이 목록에 안 보이면**: Python이 설치 안 되어 있거나 PATH에 등록 안 됨.
> 사용설명서.md의 "단계 ① Python 설치" 참조.

## 8️⃣ 첫 설치 — 부품 다운로드

(이미 `설치하기.bat` 으로 설치하셨다면 이 단계 건너뛰셔도 됩니다.)

### 통합 터미널 열기

1. 메뉴 → **터미널 → 새 터미널** (또는 ``Ctrl + ` ``)
2. VSCode 하단에 검은 또는 파란 창이 나타남
3. 6번 단계에서 `property_blog` 폴더를 제대로 열었다면, 터미널은
   **자동으로 그 폴더에 위치**합니다.

### ⚠️ 먼저 — 터미널 위치 확인

부품 설치 전에 터미널이 올바른 폴더에 있는지 확인하세요.
터미널에 다음을 입력하고 엔터:

```
dir
```

화면에 `requirements.txt`, `app.py`, `src` 등이 보이면 OK입니다.

만약 안 보이면 터미널이 엉뚱한 폴더에 있는 것입니다.
아래 명령으로 `property_blog` 폴더로 이동하세요 (경로에 한글·공백이 있으니 **따옴표 필수**):

```
cd "G:\내 드라이브\0.사내공유\1.부동산_공유\블로그자동작성\property_blog"
```

> 💡 경로 입력이 번거로우면, 터미널에 `cd ` (cd + 공백)까지만 치고
> 탐색기에서 `property_blog` 폴더를 터미널 창으로 **드래그**하면 경로가 자동 입력됩니다.

### 부품 설치

올바른 폴더에 있는 것을 확인한 뒤, 터미널에 다음을 입력하고 엔터:

```
pip install -r requirements.txt
```

1~3분 기다린 후 **`Successfully installed...`** 메시지가 뜨면 완료.

> ⚠️ **"requirements.txt를 찾을 수 없습니다" 오류가 나면**: 위 "터미널 위치 확인"
> 단계를 안 한 것입니다. `dir` 로 확인하고 `cd` 로 폴더 이동 후 다시 시도하세요.

## 9️⃣ .env 파일 편집

### `.env` 파일이 없으면 만들기 (터미널 한 줄 — 가장 쉬움)

터미널에 다음을 입력하고 엔터:

```
copy .env.example .env
```

→ `.env.example` 을 복제해 `.env` 파일이 같은 폴더에 생깁니다.

> **확인**: `dir .env*` 입력 → `.env` 와 `.env.example` 둘 다 보이면 성공.

### `.env` 편집

1. VSCode 왼쪽 파일 목록에서 `.env` 를 **한 번 클릭**
2. 가운데 편집기에 파일 내용이 표시됨
3. 다음 부분들을 본인 정보로 수정:

```bash
# Claude API 키 (필수)
ANTHROPIC_API_KEY=sk-ant-실제로받은긴키...

# 사무실 공용 비밀번호 (직원에게 알려줄 비밀번호)
OFFICE_PASSWORD=japanreal2026

# 회사 정보
COMPANY_NAME=JRE일본부동산
KAKAO_TALK_ID=japanreal2
COMPANY_PHONE=070-8201-5740
```

4. **`Ctrl + S`** 로 저장

> 💡 `=` 기호 양옆에 공백 없어야 하고, 값에 큰따옴표 붙이지 마세요.
> VSCode가 헷갈리면 터미널에서 `notepad .env` 로 메모장 편집도 가능합니다.

## 🔟 프로그램 실행하기

3가지 방법이 있습니다. **방법 C (터미널 명령)가 가장 안정적**이라 먼저 추천합니다.

### 방법 C: 터미널에 직접 명령 ⭐ 가장 안정적

VSCode 터미널에 다음을 입력하고 엔터:

```
python -m streamlit run app.py
```

> 💡 **`streamlit run app.py` 가 아니라 `python -m streamlit run app.py`** 입니다.
> 앞에 `python -m` 을 붙이면 PATH 설정과 무관하게 항상 작동합니다.
> ("streamlit 명령을 인식할 수 없습니다" 오류를 예방)

처음 한 번은 "Email:" 을 묻습니다 → **그냥 엔터** (마케팅 메일 거부).

### 방법 A: F5 한 번

미리 설정 파일을 넣어두어 F5로도 실행됩니다.

1. 왼쪽 파일 목록에서 **`app.py`** 를 먼저 클릭해서 연다
   (`.env` 가 열린 상태에서 F5를 누르면 엉뚱한 안내가 뜹니다)
2. 키보드 **`F5`** 누르기
3. 화면 위쪽에 모드 선택지가 나옴:
   - **📱 단독 사용 모드 (이 PC에서만)** ← 혼자 사용 시
   - **🏢 사무실 공용 모드 (직원 PC에서 접속 가능)** ← 공유 시
4. 원하는 모드 선택 → 브라우저가 자동으로 열림

> ⚠️ **"dotenv 디버깅 확장이 없습니다" 안내가 뜨면**: `.env` 파일이 열린 채로
> F5를 누른 것입니다. 안내창은 "취소"를 누르고, `app.py` 를 먼저 연 뒤
> 다시 F5를 누르거나 방법 C(터미널 명령)를 사용하세요.

### 방법 B: 좌측 사이드바 "실행 및 디버그"

1. 왼쪽 사이드바의 **▶ 모양 아이콘** 클릭 (재생 + 벌레 모양)
2. 위쪽 드롭다운에서 모드 선택
3. 옆의 **녹색 ▶ 버튼** 클릭

## 1️⃣2️⃣ 자주 쓰는 기능

### 🔥 자동 저장 (강력 추천)

매번 `Ctrl + S` 안 눌러도 자동으로 저장:

- 메뉴 → **파일 → 자동 저장** 클릭

### 🔥 단축키 모음

| 단축키 | 기능 |
|-------|------|
| `F5` | 프로그램 실행 |
| `Ctrl + S` | 파일 저장 |
| `Ctrl + \`` | 터미널 열기/닫기 |
| `Ctrl + B` | 왼쪽 파일 목록 보이기/숨기기 |
| `Ctrl + Shift + P` | 명령 팔레트 열기 |
| `Ctrl + P` | 파일 이름으로 빠르게 열기 |
| `Ctrl + F` | 현재 파일에서 검색 |
| `Ctrl + Shift + F` | 전체 폴더에서 검색 |
| `Ctrl + ,` | 설정 열기 |
| `Ctrl + Z` | 되돌리기 |
| `Ctrl + +/-` | 글자 크기 키우기/줄이기 |

### 🔥 화면 분할 (여러 파일 동시 보기)

파일 탭을 드래그해서 오른쪽이나 아래로 끌면 화면이 분할됩니다.

### 🔥 오류 메시지 보기

빨간 글씨로 표시되는 오류는 하단 터미널에 그대로 나옵니다. **그 메시지를 그대로 복사해서** ChatGPT, Claude.ai, 또는 개발자에게 보내면 즉시 해결책을 알 수 있어요.

## 1️⃣3️⃣ 블로그 글 스타일 커스터마이징

> 이건 살짝 개발자스러운 작업이지만, 변경 자체는 어렵지 않습니다.

블로그 글의 톤이나 구조를 바꾸고 싶으시면:

### 단계

1. VSCode에서 왼쪽 파일 목록 → **`src` → `generator.py`** 더블클릭
2. `Ctrl + F` 로 검색 → **"BLOG_GENERATION_PROMPT"** 입력
3. 따옴표 안의 한국어 지시문을 수정
4. `Ctrl + S` 저장
5. 프로그램이 실행 중이면 터미널에서 `Ctrl + C` → F5 로 재시작

### 수정 예시

**예 1) 더 격식 있는 톤으로**
```python
# 원문:
6. **친근하고 따뜻한 톤** — "안녕하세요, 도쿄 ○○부동산입니다" 식으로 시작

# 수정 후:
6. **전문적이고 격식 있는 톤** — "안녕하십니까, 본 ○○부동산입니다" 식으로 시작
```

**예 2) 이모지 없이 깔끔하게**
```python
# 원문:
- <h2>📍 매물 기본정보</h2> (표 형태)

# 수정 후:
- <h2>매물 기본정보</h2> (표 형태, 이모지 사용 자제)
```

**예 3) 비자별 추가 안내문구**
```python
# 추가 지시 삽입:
- 워홀러 타깃 글 작성 시, 일본 도착 후 1주일 내 처리해야 할 행정 절차(マイナンバー, 은행 개설 등)도 한 줄로 언급
```

### 검토

수정 후 실제로 글을 한 번 생성해보고 톤을 확인하세요. 마음에 안 들면 다시 수정 (반복).

## 1️⃣4️⃣ 문제가 생기면

### "Python: Select Interpreter" 에 Python이 안 보임
→ Python이 PATH에 추가 안 된 상태. Python 설치 프로그램을 다시 실행 →
"Modify" → "Add Python to environment variables" 체크 → Install →
VSCode를 완전히 껐다 켜기.

### "requirements.txt를 찾을 수 없습니다"
→ 터미널이 `property_blog` 폴더가 아닌 다른 곳에 있음. 터미널에 `dir` 입력해서
`requirements.txt` 가 보이는지 확인하고, 안 보이면 폴더 이동:
```
cd "G:\내 드라이브\0.사내공유\1.부동산_공유\블로그자동작성\property_blog"
```

### 터미널에 "streamlit: 명령을 인식할 수 없습니다" / "is not recognized"
→ streamlit이 PATH에 등록 안 됨. `streamlit run app.py` 대신 아래 명령 사용:
```
python -m streamlit run app.py
```

### F5를 눌렀더니 "dotenv 디버깅 확장이 없습니다" 안내가 뜸
→ `.env` 파일이 열린 상태에서 F5를 누른 것. 안내창은 "취소" 누르고,
왼쪽에서 `app.py` 를 먼저 클릭해 연 뒤 F5를 누르거나, 터미널에서
`python -m streamlit run app.py` 실행.

### F5를 눌렀는데 "디버그 구성 선택" 화면이 안 뜸
→ `.vscode/launch.json` 파일이 없는 상태. 압축 파일을 다시 풀거나,
터미널 방식(`python -m streamlit run app.py`)으로 실행하세요.

### "ModuleNotFoundError: No module named 'anthropic'"
→ 부품 설치가 안 됨. 터미널 위치 확인(`dir`) 후 `pip install -r requirements.txt` 실행.

### "AuthenticationError" / "Invalid API key"
→ `.env` 파일의 API 키 오류. 키 앞뒤 공백·따옴표 제거, 키 전체가 복사됐는지 확인.

### Google Drive 동기화 아이콘이 계속 도는데 프로그램이 느림
→ `pip install` 등 무거운 작업 중에는 정상입니다. 작업이 끝나면 잦아듭니다.
계속 심하게 느리면 Google Drive를 잠시 "동기화 일시중지" 했다가 작업 후 재개하세요.

### VSCode가 무거워서 느림
→ 사양이 낮은 PC면 그냥 `.bat` 파일 더블클릭으로 사용해도 됩니다. VSCode가 필수는 아니에요.

### .env 편집 후 변경이 적용 안 됨
→ 프로그램을 재시작해야 합니다. 터미널 `Ctrl + C` 로 종료 후 다시 실행.

### 터미널이 한국어 깨짐
→ 터미널 우측 상단 톱니바퀴 → "Configure Terminal Settings" → 인코딩 UTF-8 설정.

## 🎯 한눈에 보는 일상 워크플로 (VSCode 사용 시)

```
[ 첫 1회 설치 ]
  ① VSCode 설치 → Korean Pack + Python 확장
  ② property_blog 폴더 열기
     (G:\내 드라이브\...\블로그자동작성\property_blog)
  ③ Python 인터프리터 선택
  ④ 터미널에서 dir 로 위치 확인 → pip install -r requirements.txt
  ⑤ 터미널에서 copy .env.example .env → .env 편집 (API 키, 비밀번호)

[ 매일 사용 ]
  ① VSCode 실행 → 최근 폴더에서 property_blog 클릭
  ② 터미널에 python -m streamlit run app.py 입력
  ③ 브라우저에서 작업
  ④ 종료 시 터미널에서 Ctrl + C

[ 설정 변경 시 ]
  - .env 파일 편집 → 저장 → 재시작
  - 글 스타일 변경 시 src/generator.py 또는 styles/ 폴더 편집
```
