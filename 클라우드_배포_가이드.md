
## ⚠️ 시작 전 — 가장 중요한 보안 주의사항

**API 키가 들어있는 파일은 절대 GitHub에 올리면 안 됩니다.**

다음 두 파일은 업로드에서 **제외**해야 합니다:

- `.env` (직접 만드신 경우)
- `.streamlit/secrets.toml` (직접 만드신 경우)

> 💡 `.env.example` 와 `.streamlit/secrets.toml.example` 는 **예시 템플릿**이라
> 올라가도 안전합니다 (`.example` 이 붙은 것). 실제 키가 든 파일만 위험합니다.

이 가이드에서는 처음부터 안전하게 진행하니, 위 두 파일을 폴더에서
**미리 삭제하거나, 업로드 시 선택하지 않으면** 됩니다.

## ① GitHub 비공개 저장소 만들기

저장소(Repository) = GitHub에서 프로젝트 파일을 보관하는 공간입니다.

### 단계

1. 브라우저에서 https://github.com 접속 → 로그인
2. 화면 오른쪽 위 **`+`** 아이콘 클릭 → **"New repository"** 선택
3. 저장소 정보 입력:
   - **Repository name**: `jreblog` (또는 원하는 이름)
   - **Description**: `JRE일본부동산 블로그 자동작성 시스템` (선택)
   - **Public / Private**: ⚠️ 반드시 **"Private"(비공개)** 선택
   - "Add a README file" 등 나머지 체크박스는 **모두 비워두기**
4. 아래 **"Create repository"** 클릭

→ 빈 저장소가 만들어집니다. 다음 단계에서 파일을 채웁니다.

---

## ② 프로그램 파일을 저장소에 올리기

가장 쉬운 방법은 **브라우저에서 드래그앤드롭**입니다 (프로그램 설치 불필요).

### 준비 — 올리면 안 되는 파일 확인

`property_blog` 폴더를 Windows 탐색기로 열고, 다음 파일이 있으면 **삭제**하세요
(없으면 그냥 넘어가세요):

- `.env`
- `.streamlit/secrets.toml`

> 숨김 파일이 안 보이면: 탐색기 상단 **"보기" → "숨김 항목"** 체크

남아 있어야 하는 것 (안전한 템플릿):
- ✅ `.env.example`
- ✅ `.streamlit/secrets.toml.example`
- ✅ `.streamlit/config.toml`
- ✅ `.gitignore`

### 업로드

1. 방금 만든 GitHub 저장소 페이지에서
   **"uploading an existing file"** 링크 클릭
   (또는 **"Add file" → "Upload files"**)
2. Windows 탐색기에서 `property_blog` 폴더를 **열고**,
   폴더 **안의 모든 파일과 폴더를 전체 선택**(`Ctrl + A`)
3. 선택한 것들을 GitHub 업로드 영역으로 **드래그앤드롭**
   - ⚠️ `property_blog` 폴더 자체가 아니라, 그 **안의 내용물**을 올립니다
   - `app.py`, `requirements.txt`, `src` 폴더 등이 저장소 최상위에 와야 함
4. 파일들이 올라가면 화면 아래 **"Commit changes"** 클릭

### 확인

저장소 메인 화면에 다음이 보이면 성공:
```
app.py
requirements.txt
src/
styles/
.streamlit/
...
```

> 💡 `.gitignore` 가 포함돼 있으면, 혹시 `.env` 를 실수로 올려도
> 이후 Git이 추적하지 않도록 막아줍니다. 하지만 처음부터 안 올리는 게 가장 안전합니다.

## ③ Streamlit Community Cloud 가입

1. 브라우저에서 https://share.streamlit.io 접속
2. **"Continue with GitHub"** 클릭
3. GitHub 계정으로 로그인 → 권한 허용
4. Streamlit이 GitHub 저장소에 접근하도록 승인
   - 비공개 저장소도 배포하려면 권한 승인이 필요합니다
   - "Authorize Streamlit" 클릭

---

## ④ 앱 배포 + 비밀정보 입력

### 배포 시작

1. Streamlit 대시보드에서 **"Create app"** (또는 "New app") 클릭
2. **"Deploy a public app from GitHub"** 계열 옵션 선택
3. 배포 정보 입력:
   - **Repository**: `본인계정/jreblog` 선택
   - **Branch**: `main` (기본값)
   - **Main file path**: `app.py`

### ⭐ 비밀정보(Secrets) 입력 — 가장 중요

배포 화면에서 **"Advanced settings"** 클릭 → **"Secrets"** 칸이 나옵니다.

이 칸에 아래 내용을 **붙여넣고, 실제 값으로 수정**하세요:

```toml
ANTHROPIC_API_KEY = "sk-ant-api03-실제키를여기에"
OFFICE_PASSWORD = "japanreal2026"
COMPANY_NAME = "JRE일본부동산"
KAKAO_TALK_ID = "japanreal2"
COMPANY_PHONE = "070-8201-5740"
```

> 💡 이 Secrets 칸은 GitHub와 무관하게 Streamlit 안에만 안전하게 저장됩니다.
> API 키가 GitHub에 노출되지 않습니다.

- **Python version**: 3.12 (기본값 그대로)

### 배포 실행

**"Deploy"** 클릭 → 2~5분간 자동으로 설치·배포가 진행됩니다.
화면에 로그가 흐르다가 앱 화면이 뜨면 성공입니다.

## ⑤ 주소를 jreblog 로 변경

배포 직후 주소는 임의의 이름입니다. 이것을 `jreblog`로 바꿉니다.

1. 앱 화면 오른쪽 위 또는 대시보드에서 해당 앱의 **"⋮" (점 3개) → "Settings"**
2. **"General"** 탭
3. **"Subdomain"** (또는 "Custom subdomain") 칸에 **`jreblog`** 입력
4. 저장

→ 최종 주소: **`https://jreblog.streamlit.app`**

> 서브도메인은 최소 6자 이상이어야 합니다. `jreblog`는 7자라 OK입니다.

## ⑥ 접속 테스트

1. 브라우저에서 **`https://jreblog.streamlit.app`** 접속
2. 🔐 비밀번호 입력 화면이 뜸 → ④에서 정한 `OFFICE_PASSWORD` 입력

4. 정상 작동하면 **배포 완료!** 🎉

## 🔄 나중에 코드를 수정하면? (자동 반영)

배포 후 블로그 스타일 등을 수정하고 싶을 때:

1. GitHub 저장소에서 해당 파일을 열고 수정 (연필 아이콘 → 편집 → Commit)
   - 또는 수정한 파일을 다시 "Upload files"로 덮어쓰기
2. **Streamlit이 자동으로 변경을 감지해 앱을 다시 배포합니다** (1~2분)
3. 별도 작업 불필요

> 💡 API 키·비밀번호 변경은 코드가 아니라 Streamlit **Settings → Secrets**에서
> 수정합니다.

## 🆘 문제 해결

### 배포 중 "Error installing requirements"
→ `requirements.txt` 가 저장소 최상위에 있는지 확인.
   `app.py` 와 같은 위치에 있어야 합니다.

### 앱은 떴는데 "API key" 관련 에러
→ Settings → Secrets 에 `ANTHROPIC_API_KEY` 가 정확히 입력됐는지 확인.
   따옴표 포함, 키 전체가 들어갔는지 점검.

### "This app has gone to sleep"
→ 무료 플랜은 일정 시간 미사용 시 휴면 상태가 됩니다.
   화면의 **"Yes, get this app back up!"** 버튼을 누르면 30초~1분 후 깨어납니다.
   정상 동작이며, 직원이 자주 쓰면 거의 항상 깨어 있습니다.

### 도면 5개 동시 처리가 느림
→ 무료 플랜은 메모리 1GB 제한이 있습니다. 느리면 2~3개씩 나눠서 처리하세요.

### 비공개 저장소가 Streamlit에 안 보임
→ Streamlit 대시보드 → 우측 상단 계정 → "Settings" → GitHub 권한을
   다시 확인하고 저장소 접근을 승인하세요.

### 서브도메인 jreblog 가 이미 사용 중이라고 나옴
→ 다른 이름으로 변경 (예: `jre-japan-blog`, `japanreal-blog`).
